# 纯水制备优化装置网页模型开发完整需求说明

## 一、项目整体定位

本项目为基于Web的纯水制备优化装置仿真监控模型，对应实体装置为「一种基于树脂软化-余压回收-浓盐循环的纯水制备优化装置」，核心目标是通过前端技术实现无传感器接入的模拟监控与控制功能。

- 视觉风格：参考中国移动界面设计逻辑，采用上绿下白配色。
- 技术栈：推荐HTML + CSS + JavaScript（原生），可选轻量框架Vue.js，图表可视化可使用ECharts/Chart.js，纯前端实现数据模拟与交互。
- 核心基准：所有参数以40吨/小时（m³/h）进水流量为核心基准值，关联参数按工业逻辑同步波动。

## 二、页面布局与视觉规范

### 1. 整体布局结构

| 区域 | 背景色 | 核心内容 |
| --- | --- | --- |
| 顶部栏 | 绿色 | 展示项目标题：「一种基于树脂软化-余压回收-浓盐循环的纯水制备优化装置」 |
| 中部主体区 | 白色 | 上半部分：5个模拟仪表盘（数据监测）；下半部分：3大功能模块（控制+展示） |

### 2. 配色规范

- 顶部栏：背景色建议 #008A4B（中国移动风格绿色），文字为白色。
- 主体区：背景白色，模块边框/分隔线用浅灰色（#E5E5E5），按钮/高亮元素用绿色系。

## 三、核心功能模块（含40m³/h基准数据）

### （一）顶部仪表盘数据监测模块

标题下方横向排列5个模拟仪表盘（圆形指针/数字卡片形式），所有参数以40m³/h进水流量为基准，数值自动波动且保留1-2位小数：

| 序号 | 参数名称 | 单位 | 数值范围 | 与40m³/h进水流量的关联逻辑 |
| --- | --- | --- | --- | --- |
| 1 | 原水流量 | m³/h | [38, 42] | 基准值40，每5-10秒±2内随机波动 |
| 2 | 浓盐水进口压力 | MPa | [0.6, 0.75] | 流量40→压力0.7MPa，流量±2→压力±0.02MPa |
| 3 | 余压发电机功率 | kW | [18, 22] | 流量40→功率20kW，流量±2→功率±1kW |
| 4 | 浓盐循环流量 | m³/h | [19, 21] | 固定为进水流量的50%左右，同步±1波动 |
| 5 | 蓄电池储能 | kWh | [35, 45] | 充电时从35升至45（约300秒），放电时反之 |

### （二）左侧设备控制中心模块

点击子模块入口可进入弹窗/子页面进行参数设置，所有可设置范围适配40m³/h进水流量工况：

#### 1. 一级反渗透模块

- 可设置项：
  - 运行压力（MPa）：[1.8, 2.2]（基准值2.0），输入框/滑块调节
  - 回收率（%）：[65, 72]，输入框/滑块调节
  - 启停状态：开关按钮（启动后浓盐水压力±0.03MPa波动）
- 反馈：设置后实时显示当前配置值。

#### 2. 二级反渗透模块

- 可设置项：
  - 运行压力（MPa）：[1.2, 1.6]（基准值1.4），输入框/滑块调节
  - 回收率（%）：[80, 85]，输入框/滑块调节
  - 启停状态：开关按钮（启动后浓盐循环流量±0.5m³/h波动）
- 反馈：设置后实时显示当前配置值。

#### 3. 过滤器模块

- 可设置项：
  - 过滤压差（MPa）：[0.1, 0.15]（模拟显示，适配40m³/h流量）
  - 滤芯寿命（%）：初始90%，每30秒递减1%，可点击「更换滤芯」重置为100%
  - 冲洗周期（h）：[12, 48]，输入框设置
- 反馈：实时显示压差、滤芯寿命，更换后刷新状态。

#### 4. EDI系统模块

- 可设置项：
  - 电流（A）：[5, 7]（基准值6A），输入框/滑块调节
  - 产水电阻率（MΩ·cm）：[17.5, 18.2]（随电流变化模拟显示）
  - 启停状态：开关按钮（启动后蓄电池放电速率+1%/10秒）
- 反馈：设置后显示当前配置与模拟产水指标。

### （三）右上树脂再生盐投料系统模块

无需跳转，直接在当前区域展示状态+手动控制，参数适配40m³/h流量：

#### 1. 再生盐储存量

- 展示：数值 + 进度条，单位kg，范围[0, 800]（初始值600kg）。
- 模拟逻辑：手动投料后减少对应数值，减少量≤剩余储量。

#### 2. 离子交换树脂健康状态

- 展示：文字标签 + 状态图标（绿色「健康」/黄色「需再生」/红色「失效」）。
- 模拟逻辑：
  - 健康：盐储量≥200kg 且 运行时间＜2小时；
  - 需再生：盐储量＜200kg 或 运行时间≥2小时；
  - 失效：盐储量=0 且 需再生状态持续≥30分钟。

#### 3. 手动投盐控制

- 交互：输入框（范围[0, 剩余储量]）+「确认投料」按钮；
- 约束：输入值超剩余储量时弹窗提示，投料后树脂状态重置为「健康」。

### （四）右下蓄电池管理系统模块

无需跳转，直接展示状态+手动控制，参数适配40m³/h流量：

#### 1. 充放电状态

- 展示：文字标签（「充电中」/「放电中」/「待机」）+ 对应图标。
- 模拟逻辑：
  - 充电中：余压发电机功率≥18kW 且 电量＜100%；
  - 放电中：反渗透/EDI启动 且 电量＞0%；
  - 待机：无负载且电量=100% 或 禁用状态。

#### 2. 电量百分比

- 展示：圆形进度条 + 百分比数值（%），范围[0, 100]。
- 模拟逻辑：
  - 充电中：每10秒+1%；
  - 放电中：每10秒-1.5%；
  - 待机：数值不变。

#### 3. 手动控制开关

- 交互：「蓄电池启用/禁用」开关按钮；
- 逻辑：禁用后状态变为「待机」，电量停止变化；启用后恢复模拟逻辑。

## 四、开发步骤建议

1. 页面骨架搭建：用HTML完成标题栏、仪表盘区、三大功能模块的布局结构。
2. 样式美化：用CSS实现上绿下白配色，调整模块间距、边框、按钮样式等。
3. 数据模拟开发：实现仪表盘参数自动更新的核心逻辑，绑定DOM元素展示数据。
4. 交互功能开发：实现控制中心模块跳转、投盐操作、蓄电池开关等交互逻辑。
5. 联调优化：测试所有参数联动逻辑，确保数据合理性与交互流畅性。

### 总结

1. 模型整体采用「上绿下白」视觉风格，所有参数以40m³/h进水流量为基准，关联参数按工业逻辑同步波动，而非独立随机值。
2. 核心功能分为仪表盘数据监测、设备控制中心、树脂投料系统、蓄电池管理四大模块，覆盖状态展示与手动控制需求。
3. 开发需按「布局-样式-数据-交互-优化」的步骤推进，确保功能完整且交互流畅。

## 五、当前项目运行方式（Next.js + Electron 双模式）

本项目已升级为「Linux 开发联调 + Windows 打包发布」模式。

### 1. 本地开发（Linux / Windows 都可）

```bash
npm install
npm run dev
```

- 仅启动 Next.js 开发服务（3000端口），用于纯前端联调。

如需在开发阶段联调 Electron 壳层，可选：

```bash
npm run dev:electron
```

Linux 受限环境可用：

```bash
npm run dev:electron:linux
```

### 2. 生产构建（静态导出）

```bash
npm run build
```

- Next.js 使用 `output: 'export'` 导出静态资源到 `out/`。
- Electron 生产态会加载 `out/index.html`。

### 2.1 路由模式说明（Next.js Pages Router）

已改为文件系统自动路由（目录名决定路径）：

- 监控主页：`/`
- 一级反渗透：`/control/ro1`（文件：`pages/control/ro1/index.js`）
- 二级反渗透：`/control/ro2`（文件：`pages/control/ro2/index.js`）
- 过滤器：`/control/filter`（文件：`pages/control/filter/index.js`）
- EDI 系统：`/control/edi`（文件：`pages/control/edi/index.js`）

说明：

- 主页的「设备控制中心」点击后会进入对应路由页，而非弹窗。
- 仿真状态通过全局上下文保持，路由切换不会丢失当前模拟数据。

### 3. Windows 打包（建议在 Windows 环境执行）

```bash
npm run dist:win
```

- 会先构建 Next.js，再使用 `electron-builder` 生成 NSIS 安装包。
- 产物目录为 `release/`。

补充说明：

- 在 Linux 上直接执行 Windows 打包，若缺少 wine 依赖可能失败。
- 当前配置已关闭 Windows 可执行文件编辑步骤（`signAndEditExecutable: false`），可减少 Linux 交叉打包依赖。
- 最稳定方案仍是使用 Windows 环境或 GitHub Actions 的 `windows-latest` 执行 `npm run dist:win`。

### 4. GitHub Actions 自动打包

仓库已提供工作流：

- `.github/workflows/build-win.yml`

触发条件：

- 推送到 `main` / `master`
- 手动 `workflow_dispatch`

工作流会在 `windows-latest` 上执行 `npm run dist:win` 并上传 `release/` 产物。
